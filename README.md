# Transformer Futures Trading System

本プロジェクトは、日経225先物を対象とした、Transformerモデルによるトレンド予測フィルターと価格モメンタムを組み合わせた自動売買システムの研究・開発プラットフォームです。

## 1. 外部仕様 (External Specification)

### 1.1 システムの全体像
本システムは、以下の3つの主要フェーズで構成されます。
1.  **Feature Generation:** 30秒足のバーデータから学習用特徴量と目的変数（効率比）を生成。
2.  **Walk-Forward Training:** 過去の一定期間で学習し、次の期間で検証・テストするプロセスを繰り返す堅牢な学習。
3.  **Hybrid Backtesting:** AIの「トレンド発生予測」をフィルターとし、生の価格の「モメンタム」をトリガーとする戦略シミュレーション。

### 1.2 コマンドライン・インターフェース

#### 1.2.1 特徴量生成 (`generate_features.py`)
```bash
python -m train.entry.generate_features
````

* **入力:** `data/bars/` 配下の30秒足Parquet群
* **出力:** `data/features/` 配下の特徴量済みParquet群
* **処理:** データの定常化（対数差分）、テクニカル指標の計算、目的変数（2時間後の効率比）の算出。

#### 1.2.2 ウォークフォワード学習 (`train.py`)

```bash
python -m train.entry.train --train-days 120 --valid-days 20 --test-days 1 --start 2019-12-24
```

* **主要引数:**

  * `--train-days`: 学習に使う日数（デフォルト120日）。
  * `--start`: 処理を開始するテスト対象日。
* **出力:** 学習済みモデル（`.pth`）、学習ログ（AUC、エッジ向上率など）。

#### 1.2.3 戦略バックテスト (`backtest.py`)

```bash
python -m train.entry.backtest --start 2019-12-24 --prob-threshold 0.75 --sl-ticks 20 --tp-ticks 50
```

* **主要引数:**

  * `--prob-threshold`: AIが「トレンドが出る」と確信する確率の閾値。
  * `--sl-ticks / --tp-ticks`: 損切り・利食い幅（1tick=5円）。
* **出力:** 損益統計（プロフィットファクター、最大ドローダウン）、資産曲線（`equity_curve.png`）。

## 2. 内部詳細設計 (Internal Design)

### 2.1 特徴量エンジニアリング (`features.py`)

本システムは、Transformerが相場の微細な変化とマクロな構造を同時に理解できるよう、多様な観点から特徴量を抽出しています。

#### 1. 価格・ボラティリティ特徴量

* **対数リターンと複数時間足のモメンタム:** 1分〜60分の複数時間窓（`BAR_1M`〜`BAR_60M`）における対数リターンと単純リターン。
* **実現ボラティリティ:** 5分、10分、30分のローリング標準偏差。
* **Garman-Klass 近似ボラティリティ:** OHLC（始値・高値・安値・終値）をフル活用し、単なる終値ベースよりも高精度なボラティリティを算出。
* **バー内位置 (Close Position in Bar):** その足の中で、終値が高値と安値の間のどこに位置するか（ヒゲの長さや方向性を表現）。

#### 2. 出来高・約定フロー特徴量（Order Flow）

* **売買比率 (Buy/Sell Ratio):** 歩み値（Tick Test）から復元された買い約定と売り約定の偏り（Volume Pressure）。
* **累積ネットボリューム:** 直近数分間の `(買い出来高 - 売り出来高)` の累積値（`signed_volume_sum`）により、買い圧力・売り圧力の蓄積を検知。
* **Tick 密度・スピード:** 出来高あたりのTick数や、直近の平均Tick速度との比率（`tick_speed_ratio`）。アルゴリズム取引の介入や閑散相場を識別します。
* **VWAP乖離率:** 30秒VWAP、15分VWAP、およびその日のセッション開始からの累積VWAP（`session_vwap`）からの価格乖離。

#### 3. クロスアセット特徴量（外部指標との連動性）

S&P500、NASDAQ、DOW、Gold、WTI原油などの MT5 外部指標を非同期結合（Forward Fill）し、国際市場との連動性を評価します。

* **米国株リスクオン指標:** 米3大指数のリターンを合成したリスクオン/オフの強さ（`us_risk_on_ret`）や、指数間のばらつき（`us_index_dispersion`）。
* **相対強弱と遅行 (Lead-Lag):** 日経225のリターンとS&P500のリターン差（`rel_strength`）や、過去のS&P500の動きに対する追随遅れ（`lead_lag_vs_sp500_1m_lag1`等）。
* **ショック・ダイバージェンス:** 株価指数と安全資産（Gold）、あるいはコモディティ（原油）の動きが乖離した瞬間を検知（`gold_vs_sp500_divergence`）。

#### 4. カレンダー・セッション特徴量

* **時間帯の三角関数エンコーディング:** `minute_of_day` や `weekday` を `sin` / `cos` 波に変換し、時間の循環性（周期性）をTransformerに学習させます。
* **セッション固有フラグ:** 寄り付き直後（`is_opening_window`）、昼休み前（`is_lunch_edge`）、深夜（`is_night_late`）など、特定の流動性イベントが発生しやすい時間帯を明示。

#### 5. データの定常化 (Stationarity) とリーク防止

* **1階差分による定常化:** ディープラーニングモデルの過学習と発散を防ぐため、`open`, `high`, `low`, `close`, `vwap` などの「生の絶対価格（非定常データ）」は、すべて1つ前の行との差分（1階差分）に変換（`make_prices_stationary`）してからモデルに入力します。
* **絶対価格の保護:** バックテストにおけるシミュレーション誤差（オフバイワンエラーや累積和によるギャップの歪み）を排除するため、定常化を行う直前に生の絶対価格を `raw_*_abs` としてバックテスト専用に保持します。

#### 6. 効率比 (Efficiency Ratio) ラベル

* **効率比 (Efficiency Ratio) ラベル:**

  * `label_efficiency_240` (2時間先): 「価格の移動距離 ÷ 走行距離」で算出。
  * ノイズの多い「乱高下」と、利益を出しやすい「一直線のトレンド」を分離するための指標。

### 2.2 データセットと動的ラベリング (`dataset.py`)

* **Z-Score Normalization:** 学習データ（Train）の平均と標準偏差を算出し、検証・テストデータに適用。未来情報の混入（Data Leakage）を厳密に排除。
* **動的閾値による2値分類:**

  * 効率比の生の値は相場環境によって分布が変わるため、固定の閾値を使いません。
  * 学習期間内の効率比の「上位20%」を動的に閾値として算出し、それを超えるものを正例（1）、それ以外を負例（0）としてラベリングします。これにより、モデルは常に「その時々の相場における最高品質のトレンド」を学習します。

### 2.3 モデルアーキテクチャ (`model.py`)

* **Time Series Transformer:**

  * **Positional Encoding:** 時系列の「順序」情報を保持。
  * **Multi-Head Attention:** 過去のシーケンス（デフォルト60本=30分）内の複数の相関関係を同時に注視。
  * **出力層:** シグモイド関数前段階のロジットを出力。

### 2.4 戦略ロジックとリスク管理 (`backtest.py`)

* **AIフィルター (Filter):** Transformerの予測確率が `prob_threshold` を超えた場合のみ、その瞬間に「トレンドが発生する窓」が開いたと判定。
* **モメンタムトリガー (Trigger):** AIが「窓」を開けた瞬間、直近5分間の価格変化がプラスなら「買い」、マイナスなら「売り」でエントリー。遅行指標（SMAクロス等）を排除した鋭い反応を実現。
* **クールダウン期間 (Cooldown):** 決済後、一定期間（例：30分間）は再エントリーを禁止。マシンガン・エントリー（連続損切り）によるコスト負けを物理的に遮断。
* **非対称SL/TP管理:** `high / low` データを参照し、1分足の内部で損切り・利食いに到達したかをミリ秒精度に近いロジックでシミュレート。
